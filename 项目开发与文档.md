## 核心模块详解

### 1. DMEngine (`dm_engine.py`)

**职责**：游戏状态机、AI 模型调用、流程控制

**关键状态变量**：

| 变量 | 说明 |
|------|------|
| `game_phase` | 当前阶段（opening/discussion/vote/owner_confrontation/ending） |
| `designated_speaker` | 当前被点名的玩家 |
| `released_clues` | 已公开线索列表 |
| `search_system_enabled` | 搜证系统是否开启 |
| `found_evidence` | 已发现证据 |
| `owner_confrontation_turns` | 业主对峙轮次计数 |

**核心方法**：

| 方法 | 功能 |
|------|------|
| `start_game()` | 五步开场白流程 |
| `chat(user_input)` | 处理玩家输入，返回 DM 回复 |
| `start_vote()` | 进入最终公开答案阶段 |
| `release_clue(clue_id)` | 公开指定线索 |
| `save_game()` / `load_game()` | 存档/读档 |
| `poll_timed_events()` | 检查定时事件（线索公开、搜证开启） |

**AI 控制标记**：

DM 模型输出需包含 6 个控制标记：
```
DM_ACTION: SPEAK / SILENT
NEXT_SPEAKER: 狼/女巫/吸血鬼/木乃伊/ALL/UNCHANGED
PHASE_UPDATE: OPENING/DISCUSSION/VOTE/ENDING/UNCHANGED
INPUT_EVAL: ON_TOPIC/LIGHT_OFFTOPIC/HARD_OFFTOPIC/JAILBREAK/RULES_QUESTION
PROGRESS_SIGNAL: PROGRESS/STALLED/BREAKTHROUGH/UNCHANGED
MEMORY_UPDATE: JSON 格式的结构化记忆
```

### 2. ScriptData (`script_data.py`)

**职责**：存储剧本的静态数据

**数据结构**：

```python
SCRIPT_DATA = {
    "meta": {...},           # 元数据
    "shared_rules": {...},   # 共享规则
    "final_answer_check": {...},  # 最终答案检查
    "world": {...},          # 世界观
    "public_intro": {...},   # 公开介绍
    "roles": {...},          # 角色数据
    "clues": {...},          # 线索
    "solution": {...},       # 真相
    "timeline": [...],       # 时间线
    "search_system": {...},  # 搜证系统配置
}
```

---

### 3. TTSEngine (`tts_engine.py`)

**职责**：调用 ECNU TTS API 合成语音

**场景配置**：

| 场景 | 声音 | 语速 | 用途 |
|------|------|------|------|
| `dm` | xiayu | 1.0 | DM 旁白 |
| `owner` | liwa | 1.15 | 业主愤怒 |
| `owner_forgiveness` | liwa | 0.85 | 业主原谅 |
| `clue` | xiayu | 0.9 | 线索公开 |
| `ending` | xiayu | 0.85 | 结局演绎 |
| `ending_happy` | xiayu | 0.9 | Happy Ending |

---

### 4. BGMEngine (`bgm_engine.py`)

**职责**：场景背景音乐播放

**支持的场景**：

- opening, discussion, search, suspense, confrontation
- ending, happy_ending, bad_ending, silence

## 扩展指南

### 添加新剧本

1. 在 `script_data.py` 中创建新的数据结构
2. 修改 `DMEngine._build_system_prompt()` 加载新剧本
3. 调整结局逻辑（如需）

### 添加新命令

在 `main.py` 的 `main()` 函数中添加命令处理：

```python
if user_input.lower().startswith("/newcmd"):
    # 处理新命令
    continue
```

### 调整语音场景

1. 在 `tts_engine.py` 的 `VOICE_CONFIGS` 中添加新场景
2. 在 `main.py` 的场景检测逻辑中添加匹配规则

## 调试技巧

### 查看详细状态

```python
from dm_engine import DMEngine
dm = DMEngine()
dm.start_game()
print(dm.get_status_text())
```

### 测试问题分类

```python
dm = DMEngine()
test_inputs = ["规则是什么", "该谁发言", "有线索吗", "真相是什么"]
for inp in test_inputs:
    print(f"{inp} → {dm._classify_player_question(inp)}")
```

### 测试答案检查

```python
dm = DMEngine()
question = SCRIPT_DATA["final_answer_check"]["public_questions"][1]
print(dm._is_answer_correct(question, "孩童衣服的碎片"))
```

## 已知限制

1. API 依赖 ECNU 平台，需有效 API Key
2. 时钟机制仅在输入时检查，长时间无输入不会自动触发
3. 角色秘密信息需外部人工分发
4. 说话人识别依赖点名流程
