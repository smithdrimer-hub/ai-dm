## 扩展指南

### 添加新剧本

1. 在 `script_data.py` 中创建新的 `SCRIPT_DATA` 字典
2. 修改 `dm_engine.py` 的 `_build_system_prompt()` 加载新剧本
3. 调整结局逻辑（如答案检查规则不同）

### 添加新命令

在 `main.py` 的 `main()` 函数中添加命令处理：

```python
if user_input.lower().startswith("/newcmd"):
    # 处理新命令
    continue
```

### 调整语音场景

1. 在 `tts_engine.py` 的 `VOICE_CONFIGS` 中添加新场景
2. 在 `main.py` 的 `detect_tts_scene()` 中添加匹配规则

### 添加新情绪类别

1. 在 `mood_config.py` 的 `MOOD_PROFILES` 中添加情绪定义
2. 在 `DEFAULT_PHASE_TO_MOOD` 中添加阶段映射
3. 运行 `python download_moods_freesound.py` 下载对应音效

### 2026-04-22 - `<5` 情绪桶替代词重跑 + 软移除标记落地

目标：
- 对当前不足 5 条的情绪桶执行“替代词重跑”，不新增一级类别，且不再追求补到 10。
- 清理旧条目时尽量减少人工干预：优先用“索引剔除 + rejected 归档副本”替代硬删除。

关键改动：
- `mood_config.py`
  - 对 `regretful/triumphant/urgent/revelatory/chaotic` 进行关键词替代优化（同义词/近义词重写）。
  - 保持 20 个一级情绪桶不变，仅调整检索与匹配词。
- `bgm_engine.py`
  - 修复软移除失效问题：当 `music/moods/index.json` 存在时，仅按索引加载，不再回扫目录把已移除文件重新纳入播放池。

执行与结果：
- 管理员模式下尝试移动 rejected 文件仍报 `Access denied`（move/rename/delete 受限）。
- 采用“软移除 + 归档副本”方案：
  - 原先 orphan 17 条已复制到 `_rejected/copy_only_20260422_175629/`。
  - `<5` 桶重置移除的旧条目（2 条）已复制到 `_rejected/copy_only_bucket_reset_20260422_180143/`，并生成 `reset_manifest.csv`。
- 按替代词执行单次 CC0 回补（目标 `per-mood=5`），随后做一次完整复筛。
- 最终索引：`music/moods/index.json` 共 206 条。

移除标记与原因（本轮）：
- 标记机制：`index.json` 不再收录即视为“已移除”；物理文件保留在原目录或仅复制归档到 `_rejected`（受权限限制无法 move/delete）。
- 主要原因：
  - `bucket_lt5_reset`：对 `<5` 桶执行旧条目重置（urgent/chaotic 的旧活跃条目）。
  - 质量筛选：`duration/rms/active_ratio/head_rms` 不达标。

`<5` 桶最终数量（本轮不再补充）：
- `regretful: 1`
- `triumphant: 1`
- `urgent: 1`
- `revelatory: 1`
- `chaotic: 0`

## 已知限制

1. **API 依赖**：ECNU 平台需有效 API Key
2. **时钟机制**：Windows 下通过 `msvcrt` 非阻塞轮询实现真实静默检测（180 秒未输入触发 DM 插话）；非 Windows 回退为普通 input()
3. **角色秘密信息**：需外部人工分发，程序不负责私聊
4. **说话人识别**：依赖 DM 点名流程，无实际语音识别
5. **dm_engine.py 膨胀**：~4600 行单一文件，已规划分阶段提取 `schema_runtime.py`
