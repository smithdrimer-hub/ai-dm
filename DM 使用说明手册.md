## 🎵 背景音乐推荐

当前项目使用 `generate_bgm.py` 生成的合成测试音乐，音质较简陋。推荐从以下网站下载更好的背景音乐：

### 推荐音乐网站

| 网站 | 特点 | 推荐搜索关键词 |
|------|------|----------------|
| **耳聆网** (soundverse.net) | 中文网站，CC 协议，免版权 | "悬疑"、"紧张"、"神秘"、"温馨" |
| **FreeSound** (freesound.org) | 国际最大音效库，需注册 | "suspense ambient", "mystery background" |
| **YouTube Audio Library** | YouTube 创作者资源，完全免费 | Genre: Cinematic, Dark, Ambient |
| **FMA (Free Music Archive)** | 高质量免版权音乐 | "Atmospheric", "Dark Ambient" |

### 所需音乐清单

| 文件名 | 场景 | 推荐风格 | 时长建议 |
|--------|------|----------|----------|
| `opening.mp3` | 开场/规则说明 | 轻松友好、神秘感 | 30-60 秒循环 |
| `discussion.mp3` | 讨论阶段 | 轻柔不干扰 | 30-60 秒循环 |
| `search.mp3` | 搜证阶段 | 好奇探索 | 30-60 秒循环 |
| `suspense.mp3` | 悬疑/紧张 | 紧张悬疑 | 30-60 秒循环 |
| `confrontation.mp3` | 对峙阶段 | 激烈对峙 | 30-60 秒循环 |
| `ending.mp3` | 结局演绎 | 感人性 | 30-45 秒 |
| `happy_ending.mp3` | Happy Ending | 温暖治愈 | 30-45 秒 |
| `bad_ending.mp3` | Bad Ending | 悲伤遗憾 | 30-45 秒 |

### 使用方法

1. 下载音乐文件
2. 重命名为上述文件名（支持 `.mp3`、`.wav`、`.ogg` 格式）
3. 放入 `music/` 目录
4. 游戏中输入 `/bgm on` 开启背景音乐

## 快速开始

### 第一步：配置环境

1. 确保已安装 Python 3.13 或更高版本
2. 安装依赖：
   ```bash
   pip install -r requirements.txt
   ```
3. 在项目根目录创建 `.env` 文件：
   ```
   OPENAI_API_KEY=your_api_key_here
   OPENAI_BASE_URL=https://chat.ecnu.edu.cn/open/api/v1
   OPENAI_MODEL=ecnu-max
   ECNU_TTS_MODEL=ecnu-tts
   ECNU_DEFAULT_VOICE=xiayu
   ```

### 第二步：运行游戏

```bash
python main.py
```

程序启动后会自动播放开场白，进入五步开场流程。

## 游戏流程

```
开场白（五步） → 自由讨论 → 最终公开答案 → 业主对峙 → 结局
```

### 阶段一：开场白（五步流程）

| 步骤 | 内容 | 玩家操作 |
|------|------|----------|
| 1. 规则说明 | DM 说明核心规则 | 聆听 |
| 2. 规则确认 | DM 询问是否理解 | 输入"理解了"或提问 |
| 3. 戏剧演绎 | 业主登场指控 | 聆听氛围 |
| 4. 剧本朗读 | 背景故事介绍 | 聆听 |
| 5. 任务陈述 | 说明三道答案问题 | 聆听，准备讨论 |

### 阶段二：自由讨论

- **点名发言**：DM 点名后，下一条输入默认属于该玩家
- **线索公开**：讨论 10 分钟、20 分钟时自动公开线索
- **搜证环节**：讨论 10 分钟后开放，输入 `/search [地点]`
- **DM 回应**：普通对话 DM 保持沉默，仅在规则/流程问题时回应

### 阶段三：最终公开答案

1. 输入 `/vote` 进入投票阶段
2. **所有玩家先在纸上写下答案**
3. 所有人同时公开答案
4. 操作者按顺序录入每位角色的答案
5. 录入格式：`犯人答案/碎布答案/橱柜答案`

### 阶段四：业主对峙

- 2 轮对话收集玩家解释
- 诚实 + 道歉 → Happy Ending
- 隐瞒 + 推卸 → Bad Ending

### 阶段五：结局

根据玩家表现触发对应结局，包含完整叙事和后续故事。

## 交互规则

### 发言机制

1. DM 点名指定某位玩家发言
2. 点名后，下一条输入默认属于该玩家
3. 插话时，DM 会进入确认流程，只输入角色名完成确认

### 讨论模式

- **普通对话**：DM 保持沉默
- **规则问题**：DM 简明回答
- **流程问题**：DM 直接回答
- **线索问题**：DM 告知当前线索状态
- **剧情问题**：DM 引导推理，不剧透

### 搜证系统

- **开放时间**：讨论 10 分钟后
- **搜索命令**：`/search [地点]`
- **可搜索地点**：等候室、补给橱柜、垃圾桶、画框
- **冷却时间**：每次搜索后冷却 2 回合

## 常见问题

### Q: 语音不播放怎么办？

A: 检查：
1. API Key 是否正确（查看 `.env` 文件）
2. 网络连接是否正常
3. 电脑音量是否开启
4. 运行 `/sound test` 测试
5. 如果提示"TTS 连续失败"，请稍后重试或检查网络

**说明**：v0.8.0 已修复长文本超时问题，TTS 失败会自动切换到纯文字模式。

### Q: 如何跳过开场白？

A: 当前版本不支持跳过。如需快速测试，建议直接读档。

### Q: 搜证系统不工作？

A: 确认讨论是否已进行 10 分钟以上。可输入 `status` 查看游戏时间。

### Q: 如何触发 Happy Ending？

A: 在业主对峙环节，诚实坦白并表达歉意。

## 故障排除

| 问题 | 解决方案 |
|------|----------|
| API 调用失败 | 检查网络连接，确认 API Key 有效 |
| TTS 超时失败 | v0.8.0 已修复（增加超时至 60 秒 + 文本分割），自动降级到纯文字 |
| 语音播放卡顿 | 关闭语音输出 `/sound off`，纯文字模式 |
| 游戏卡死 | 输入 `reset` 重置游戏 |
| BGM 不播放 | 确认 music/ 目录下有音频文件，或使用 `python generate_bgm.py` 生成测试音乐 |
| BGM 音质差 | 下载更好的音乐替换（见上方"背景音乐推荐"章节） |
