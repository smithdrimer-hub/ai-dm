## 一、当前架构

```
dm_engine.py (2153行)  ── 核心游戏状态机 + LLM 调用 + 引导系统
    ├── 79 个 schema delegation wrapper → schema_runtime.py (1414行)
    │     └── 阶段管理 / 线索释放 / 行动结算 / 剧透防护 / 公共知识边界
    │
    ├── main.py (843行) ── CLI 入口 + 4 引擎整合
    │     ├── TTS 引擎 (tts_engine.py)
    │     ├── BGM 引擎 (bgm_engine.py, 206 曲目 20 情绪桶)
    │     └── SFX 引擎 (sfx_engine.py, 137 条 6 分类)

          ├── detective_game_engine.py ── 单人侦探游戏运行时
```

### 2.1 游戏流程（完整闭环）

| 功能 | 状态 | 说明 |
|------|------|------|
| 开场接待 (规则说明) | ✅ | Schema 模式：4 段分段 `[1/3]规则 → [2/3]氛围 → [3/3]任务`；Legacy 模式：一次性念稿 |
| 讨论阶段 | ✅ | 自由讨论，DM 默认沉默，点名发言 + 插话确认 |
| 线索公开 | ✅ | 手动 `/clue1` `/clue2` 或定时自动（10 分/20 分） |
| 搜证系统 | ✅ | `/search [地点]`，讨论 10 分钟后开放，冷却 2 轮 |

| 业主对峙 | ✅ | 2 轮对话收集，诚实度判断（关键词 + AI 辅助） |

| 存档/读档 | ✅ | JSON 完整序列化，Schema 状态回填，向后兼容 |

## 三、功能边界（当前不做的事）

| 边界 | 原因 |
|------|------|
| 不解析 PDF/DOCX 剧本 | 文本导入仅支持 Markdown/TXT |
| Text→Schema 不能自动开局 | 必须人工确认版权、剧透边界、角色包和真相逻辑 |
| 不让 LLM 判定行动结算 | 行动/表单结算由确定性代码执行 |
| 不把单人侦探混入 DM 引擎 | 独立产品线，不共享状态 |
| Action/Form 只实现最小子集 | MURDER/GUARD/INVESTIGATE/DECLARE/VOTE，非完整 Shadow 类自由行动 |
| 私聊分发未实现 | `/packet` 是主持人本地查看工具 |
| dm_engine.py 不再加代码 | 已达 2153 行，新功能优先加新文件 |

## 四、已知缺口

### 4.1 功能缺口

| 缺口 | 说明 |
|------|------|
| 矛盾检测（空壳） | `_check_contradictions` 记录玩家陈述但不比对矛盾 |
| 没有 `/demo` 预录演示流程 | 不支持一键自动播放预录命令序列 |
| 没有 NPC 自治系统 | NPC 仅被动问答，无行动纲领/态度/压力值 |
| 没有 AI 补位模式 | 仅支持全真人玩家 |
| 复盘个性化不足 | 基于模板，未引用玩家实际讨论内容 |
| 没有总时长预算 | 仅记录讨论用时，不比较建议时间 |
| 阶段时间感知不完整 | 不覆盖开场/投票/对峙/复盘的分段用时 |

### 4.2 音频缺口

| 关键词桶 | 库存 | 影响 |
|---------|------|------|
| ending:warm resolve | 0 | Happy Ending 无对应 SFX |
| ending:mystery resolve | 0 | 谜题解决无音效 |
| ending:relief chime | 0 | 如释重负无音效 |
| ending:ending swell | 1 | 结局高潮音效不足 |
| clue_search:photo pickup | 1 | 搜证音效单一 |
